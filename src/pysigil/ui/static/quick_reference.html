pysigil quick reference

Key controls at a glance

Use this single-page guide when navigating the pysigil preferences GUI. Each section highlights the essential actions and visual cues you will encounter.

Getting oriented

  • Provider selects the definition set to inspect. The dropdown remembers your last choice.
  • Compact toggles between a condensed view and a spacious layout for easier reading.
  • The Project field shows which project path is contributing overrides (read-only).
  • Author Tools… (author mode only) opens dialogs for editing metadata such as sections and defaults.

Working with fields

  • Hover the ℹ icon to read the field description, see the tuple-key backing the value, and preview tooltip notes.
  • The Effective value chip shows what currently applies and which scope provided it. Errors are highlighted in red.
  • Pills display every scope. A solid pill marks the active source; outlined pills hold values that are overridden; muted pills are empty.
  • Click any writable pill to focus that layer when editing. Locked scopes display a padlock cursor and ignore clicks.
  • Use Edit… to open the scoped editor. Validation feedback appears immediately.

Scope colors

Pills inherit the Aurelia palette so you can tell each layer at a glance. Only scopes that are enabled for your provider will appear.

Troubleshooting

  • Red text inside the Effective value chip indicates validation errors. Open Edit… to review details.
  • If the pill you edited stays outlined, another scope still overrides it — compare with the examples above to spot the active layer.
  • A muted pill with a dash is expected when a scope is empty. Fill it only when your workflow needs that override.
