pysigil quick reference

Configure settings with confidence

Use this page as a concise guide when navigating the pysigil preferences GUI. Each section highlights the essential actions and visual cues you will encounter.

Getting oriented

  • Provider selects the definition set to inspect. The dropdown remembers your last choice.
  • Compact toggles between condensed pills and a spacious layout for easier reading.
  • The Project field shows which project path is contributing overrides (read-only).
  • Author Tools… (author mode only) opens dialogs for editing metadata such as sections and defaults.

Working with fields

  • Hover the ℹ icon to read the field description and see the tuple-key backing the value.
  • The Effective value chip shows what currently applies and which scope provided it. Errors are highlighted in red.
  • Pills display every scope. A solid pill marks the active source; outlined pills hold values that are overridden; muted pills are empty.
  • Click any writable pill to focus that layer when editing. Locked scopes display a padlock cursor and ignore clicks.
  • Use Edit… to open the scoped editor. Validation feedback appears immediately.

Scope colors

Pills inherit the Aurelia palette so you can tell each layer at a glance. Only scopes that are enabled for your provider will appear.

Smart navigation

  • Collapse or expand sections to focus on related fields. The app remembers per-session state.
  • Use the keyboard: Tab between pills, press Space or Enter to activate focused pills.
  • Tooltips summarise values and validation issues without opening dialogs.

Troubleshooting

  • Red text inside the effective chip indicates validation errors. Open Edit… to review details.
  • Pills labelled synthetic show computed values that cannot be edited.
  • Enable the SIGIL_GUI_DEBUG environment variable before launch for verbose logging.
